Inventory rules & safeguards

Constraints that protect your stock data.

Conditional behaviors, global inventory locks, transaction protection, and reorder logic — all designed to avoid data corruption.

Protection

Global inventory lock

Cannot disable inventory if tracked items exist → “Inventory tracking cannot be disabled as inventory item(s) exist!”

Transaction lock

Track inventory toggle locked

Once item has transactions, you cannot enable/disable inventory tracking. Form rule enforced server-side.

Adjustment validation

Zero quantity / zero stock

Quantity adjustment cannot be zero; value adjustment blocked when stock on hand = 0.

Price list prerequisite

Enable in preferences first

Price Lists module only accessible if preference “Enable Price Lists” is ON, else redirected.

Import mapping

Duplicate field mapping error

Same source column cannot map to multiple system fields. Item Name, type, rate must be mapped.

Reorder notify

Two-level trigger

Requires both item reorder point AND global notify_on_reorder preference enabled.

Everything about Inventory Management

01

How to enable inventory tracking for an item?

In item form, check “Track Inventory”, then select Inventory Account and Valuation Method (FIFO/WAC). Opening stock and reorder point optional.

02

Can I import existing items with stock?

Yes, the 3-step item import supports Initial Stock and Initial Stock Rate columns. Map them during step 2.

03

What happens if I disable inventory after transactions?

System blocks it — “Inventory tracking cannot be disabled as inventory item(s) exist!” You must convert or remove tracked items first.

04

How are price lists applied in transactions?

Enable “Apply price list at line item level” in preferences, then associate price list to item. During invoice/bill, the list is applied automatically.

05

Can I adjust stock value after purchase?

Yes, use “Value Adjustment” type in Inventory Adjustments. Requires reason, account, and existing stock on hand >0.

06

Is there an audit trail for stock changes?

Every movement creates stock_movements record. Adjustments link to applied_movement_id, fully traceable.
